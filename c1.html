Compétence C1

Recenser et identifier les
ressources numériques

Déploiement de GLPI sur une VM Azure pour centraliser l'inventaire du parc informatique — machines, logiciels et utilisateurs.

// contexte

Réalisation professionnelle

Dans le cadre de mon cours de support que j'ai réalisé à l'Efrei, j'ai été amené à recenser et identifier les ressources numériques d'une organisation. Pour cela, j'ai déployé GLPI sur un serveur Azure (VM Debian/Ubuntu) afin de centraliser l'inventaire du parc informatique.

01

Déploiement du serveur GLPI sur Azure

J'ai d'abord provisionné une machine virtuelle Azure (Debian/Ubuntu) et installé GLPI 10.0.20 via le script install-glpi.sh (retrouvable dans le C17). Ce script automatise l'ensemble de la chaîne : Apache, PHP 8.3 et MariaDB.

02

Déploiement de l'agent GLPI sur les postes clients

Une fois le serveur opérationnel, j'ai installé l'agent GLPI sur chaque poste client à l'aide du script install-glpiagent.sh. L'agent remonte automatiquement l'inventaire matériel et logiciel vers le serveur.

install-glpiagent.sh 
#!/bin/bash
######################
# Launch with SUDO   #
######################

# Arrêt du script en cas d'erreur
set -e

# Installation des dépendances Perl nécessaires à l'agent
sudo apt install -y perl libnet-ssleay-perl libcrypt-ssleay-perl \
  libio-socket-ssl-perl libarchive-zip-perl \
  libwww-perl libxml-xpath-perl libyaml-tiny-perl \
  dmidecode hwdata pciutils usbutils

# Téléchargement du paquet d'installation de l'agent GLPI 1.7
wget https://github.com/glpi-project/glpi-agent/releases/download/1.7/glpi-agent-1.7-linux-installer.pl

# Installation avec connexion automatique au serveur GLPI
sudo perl glpi-agent-1.7-linux-installer.pl \
  --server="http://IP-SERVER/front/inventory.php" \
  --install --runnow

# Vérification que le service glpi-agent est bien actif
systemctl status glpi-agent

# Forcer un inventaire immédiat vers le serveur
glpi-agent --force
03

Hébergement et accès (Azure & GitHub Pages)

Le service GLPI est hébergé sur une VM Azure accessible publiquement. En parallèle, le portfolio et les documentations associées ont été déployés sur GitHub Pages pour une visibilité permanente et gratuite.

04

Consultation et export de l'inventaire

Les données remontées par les agents ont permis d'obtenir un inventaire complet accessible depuis l'interface GLPI. Trois rapports PDF ont été exportés comme preuves :

export_ordi.pdf
Liste des ordinateurs recensés (modèle, IP, OS)
export_soft.pdf
Logiciels installés sur chaque poste
export_user.pdf
Comptes utilisateurs créés et associés aux équipements

// Conclusion

Cette réalisation m'a permis de comprendre comment recenser et identifier les ressources numériques d'une organisation à l'aide d'un outil ITSM. J'ai appris à déployer un agent sur des machines Linux et à exploiter les données remontées pour constituer un inventaire fiable et centralisé du système d'information.
